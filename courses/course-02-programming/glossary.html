Programming Glossary

Essential terminology for Python programming and computational thinking.

Core Programming Concepts

Algorithm

A step-by-step procedure for solving a problem. Algorithms are the foundation of computational thinking—they transform inputs into outputs through a defined sequence of operations. In AI, algorithms range from simple sorting to complex neural network training procedures.

Variable

A named storage location for data. Variables in Python are dynamically typed, meaning their type is determined at runtime. They act as labels pointing to objects in memory, enabling code to work with data abstractly.

Data Type

A classification that specifies what kind of value a variable holds (int, float, str, bool, etc.). Data types determine what operations are valid and how values are stored in memory.

Function

A reusable block of code that performs a specific task. Functions accept inputs (parameters), process them, and optionally return outputs. They enable modular, DRY (Don't Repeat Yourself) programming.

Control Flow

The order in which statements execute. Control structures (if/else, loops) allow branching and repetition, enabling programs to make decisions and process collections of data.

Data Structures

List

An ordered, mutable collection of items. Lists support indexing, slicing, and various methods (append, pop, sort). They are Python's workhorse for sequential data storage.

Tuple

An ordered, immutable collection. Tuples are faster than lists and used for fixed collections of items. Their immutability makes them hashable and usable as dictionary keys.

Dictionary

An unordered collection of key-value pairs. Dictionaries provide O(1) average-case lookup by key, making them ideal for mappings and caching. They are implemented as hash tables.

Set

An unordered collection of unique elements. Sets support mathematical operations (union, intersection, difference) and provide O(1) membership testing.

Array (NumPy)

A fixed-type, multidimensional container optimized for numerical operations. NumPy arrays enable vectorized operations that are orders of magnitude faster than Python loops.

DataFrame (Pandas)

A two-dimensional labeled data structure with columns of potentially different types. DataFrames are the primary tool for data manipulation and analysis in Python.

Object-Oriented Programming

Class

A blueprint for creating objects. Classes define attributes (data) and methods (behavior) that their instances will have. They enable code organization and reuse through inheritance.

Object

An instance of a class. Objects combine state (attribute values) and behavior (methods). In Python, everything is an object, including functions and classes themselves.

Inheritance

A mechanism where a class (child) inherits attributes and methods from another class (parent). Inheritance enables code reuse and establishes "is-a" relationships between types.

Encapsulation

Bundling data and methods that operate on that data within a single unit, while hiding internal implementation details. Python uses naming conventions (_private, __name_mangling) for encapsulation.

Polymorphism

The ability of different classes to be treated through the same interface. In Python, duck typing enables polymorphism: "if it walks like a duck, it's a duck."

Software Engineering

Version Control

A system for tracking changes to code over time. Git is the standard, enabling collaboration, branching, and rollback. Essential for any professional development workflow.

Unit Test

A test that verifies a small, isolated piece of code (typically a function) works correctly. Unit tests form the foundation of test-driven development and continuous integration.

Debugging

The process of finding and fixing errors in code. Techniques include print statements, interactive debuggers (pdb), and systematic hypothesis testing.

Exception

An error that occurs during execution. Python's try/except mechanism allows graceful error handling, preventing crashes and enabling recovery.

Refactoring

Restructuring existing code without changing its external behavior. Goals include improving readability, reducing complexity, and enhancing maintainability.

Algorithm Analysis

Big-O Notation

A mathematical notation describing the upper bound of an algorithm's time or space complexity. O(n) means linear scaling; O(n²) means quadratic. Critical for choosing efficient algorithms.

Time Complexity

How the runtime of an algorithm scales with input size. Understanding time complexity helps predict performance and avoid bottlenecks in data-intensive applications.

Space Complexity

How memory usage scales with input size. Important when working with large datasets or memory-constrained environments.
